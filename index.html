Colors open_in_newComplete

In UI 2.0, colors are structured in two layers: primitives and tokens.

  • Primitives are the raw values. These are the new foundation and should be used only to build tokens, not applied directly in components.
  • Tokens map primitives into roles, for example text.primary, bg.surface, border.default. These tokens are defined in the Figma UI 2.0 file and must be created in code to ensure alignment between design and development.

Usage during transition

  • For UI 2.0 components, it is mandatory to use tokens.
  • For legacy UI 1.0 components, you may still apply primitives until the component is migrated to UI 2.0.
  • Once a component is transitioned to UI 2.0, replace any primitive usage with the proper token.

This ensures consistency across design and code, reduces conflicts during migration, and prepares the system for long term scalability.
